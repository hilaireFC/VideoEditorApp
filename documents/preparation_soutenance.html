❓ Questions Anticipées et Réponses Types

1. Comment avez-vous géré l'importation de vidéos et musiques depuis le téléphone ?
"J'ai utilisé la librairie react-native-image-picker. Si vous regardez dans le fichier src/screens/EditorScreen.tsx, j'ai écrit une fonction nommée handleAddVideo. Cette fonction appelle launchImageLibrary({ mediaType: 'video' }). Une fois que l'utilisateur a choisi sa vidéo, je récupère l'URL locale du fichier et je l'envoie dans mon store d'état global grâce à la fonction addVideoClip()."
2. Le système de Timeline est complexe. Comment l'avez-vous programmé de zéro ?
"Dans le fichier EditorScreen.tsx, je récupère le tableau videoClips depuis mon store Zustand. J'utilise la méthode Javascript .map() pour boucler sur ces clips. Pour chaque clip, je calcule sa durée réelle en fonction du découpage (Trim) et de sa vitesse avec la formule : (endTrim - startTrim) / speed. Cela me permet de déterminer précisément à quelle seconde démarre et s'arrête chaque bloc visuel sur l'écran par rapport au currentTime global."
3. Comment votre application gère-t-elle l'authentification des utilisateurs ?
"J'ai intégré Supabase. J'ai initialisé la connexion dans le fichier src/services/supabase.ts. Toute la logique métier est centralisée dans src/store/authStore.ts avec les méthodes login() et register() qui appellent supabase.auth.signInWithPassword(). J'utilise également AsyncStorage pour sauvegarder le token de l'utilisateur : ainsi, s'il ferme l'application, il reste connecté à la prochaine ouverture."
4. Pourquoi avez-vous utilisé Zustand plutôt que Redux pour la gestion d'état ?
"Zustand est beaucoup plus léger et performant. Dans une application de montage, les états changent des dizaines de fois par seconde (le lecteur vidéo actualise la position). Zustand m'a permis d'éviter le code 'boilerplate' de Redux. Mon fichier src/store/editorStore.ts contient tout l'état de mon projet (clips, audio, textes) de manière centralisée."
5. Je vois que vous avez un Slider (jauge) pour le volume et la vitesse. C'est le composant natif de React ?
"Non, et c'est un point technique important. Les librairies natives de sliders cassaient la compilation Gradle. Pour garantir la stabilité de l'application, j'ai développé mon propre composant Slider 100% JavaScript dans src/components/CustomSlider.tsx. J'utilise l'API PanResponder de React Native pour capter précisément les mouvements du doigt sur l'écran et je calcule le ratio de remplissage manuellement."
6. Le processus d'exportation compile-t-il vraiment la vidéo finale sur le téléphone ?
"Pour cette version de soutenance (MVP), l'écran d'exportation (ExportScreen.tsx) simule le processus de rendu. La compilation vidéo réelle sur smartphone nécessite d'intégrer FFmpeg en C++, ce qui entraînait des conflits avec l'environnement de compilation Android de la salle. Cette simulation m'a permis de valider toute l'UX lors de la démonstration, ce qui est mon objectif principal."
