📱 Guide Technique et Fonctionnement - VideoEditorApp

Ce document explique l'architecture complète de l'application que vous avez développée. Il détaille la structure des dossiers, le rôle de chaque fichier clé et le fonctionnement des composants principaux.

🏗️ 1. Architecture Globale (React Native)

L'application est construite avec React Native (TypeScript) pour garantir des performances proches du natif tout en utilisant les standards du web.

  • Frontend UI : Composants fonctionnels React avec des Hooks (useState, useEffect, useRef).
  • State Management (Gestion d'état) : Utilisation de Zustand (src/store/), choisi pour sa légèreté par rapport à Redux.
  • Backend (BaaS) : Supabase gère la base de données et l'authentification des utilisateurs.
  • Stockage Local : AsyncStorage est utilisé pour sauvegarder les sessions utilisateurs et les projets vidéos brouillons sur le téléphone.

📂 2. Structure des Dossiers

Voici l'organisation de votre code source (src/) :

  • 📁 components/ : Composants réutilisables (ex: CustomSlider.tsx).
  • 📁 navigation/ : Fichiers gérant le routage de l'app (AppNavigator.tsx).
  • 📁 screens/ : Les vues principales (Home, Editor, Login, etc.).
  • 📁 services/ : Connexions aux API externes (supabase.ts, ffmpeg.service.ts).
  • 📁 store/ : Les fichiers Zustand contenant la logique métier (authStore.ts, editorStore.ts).
  • 📁 theme/ : Couleurs, typographie et espacements globaux pour un design harmonieux.

🔐 3. Le Flux d'Authentification

L'authentification est gérée par Supabase et synchronisée dans l'application via Zustand.

  1. src/services/supabase.ts : Initialise le client Supabase avec l'URL de projet et la clé publique (Anon Key).
  2. src/store/authStore.ts : Contient les fonctions login(), register(), et logout(). Il stocke l'état global user.
  3. src/screens/LoginScreen.tsx : Appelle login(email, password) depuis le store lors du clic.
  4. src/navigation/AppNavigator.tsx : Écoute les changements d'état. Si connecté, affiche MainNavigator (Accueil), sinon affiche AuthNavigator (Connexion).

🎬 4. Le Cœur du Projet : L'Éditeur Vidéo

A. La gestion de l'état (EditorStore)

Dans src/store/editorStore.ts, l'état du projet vidéo est stocké sous forme de tableaux :

  • videoClips[] : Liste des vidéos importées.
  • audioClips[] : Liste des musiques de fond.
  • textOverlays[] : Liste des textes superposés.

B. L'Interface de Montage (EditorScreen.tsx)

C'est le fichier central (src/screens/EditorScreen.tsx).

  • Le Player : Utilise la librairie react-native-video.
  • La Timeline : Une boucle map calcule la durée active de chaque clip (endTrim - startTrim / speed) pour positionner les blocs visuellement sur un axe horizontal.
  • Les Outils : Les boutons ouvrent des Modals pour ajuster le Volume, la Vitesse, ou les Filtres.

C. Le Slider Maison (CustomSlider.tsx)

Pour éviter des bugs de compilation liés aux librairies natives, vous avez développé votre propre Slider 100% JavaScript en utilisant l'API PanResponder pour capter les glissements de doigt de l'utilisateur.

🚀 5. Le Flux d'Exportation

Dans src/screens/ExportScreen.tsx, l'écran récupère les données depuis editorStore.ts. Un algorithme simule une barre de progression réaliste à l'aide de timers (setTimeout), permettant de démontrer l'expérience utilisateur finale avec succès.
